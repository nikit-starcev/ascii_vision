# ASCII Vision — конвертация изображений в текст для LLM

Конвертирует изображение в многослойное текстовое представление: ASCII-арт + OCR + цвета + аннотации + EXIF.

## 1. Команда запуска

```bash
python -m ascii_vision.cli <путь к изображению> [флаги]
# или через консольную команду:
ascii-vision <путь к изображению> [флаги]
```

Установка (однократно):
```bash
pip install -e /home/user/LLM/ASCII_vision
# опционально: OCR и PDF
pip install -e "/home/user/LLM/ASCII_vision[ocr,pdf]"
```

## 2. Пресеты

### Схема / диаграмма (рекомендуемый)
```bash
python -m ascii_vision.cli image.png --binary --invert --multi 80 150 --colors --annotate
```

### Фотография
```bash
python -m ascii_vision.cli photo.jpg --invert -w 200 --colors
```

### Только текст (OCR)
```bash
python -m ascii_vision.cli image.png --ocr-only
```

### С авто-ориентацией (фото с телефона)
```bash
python -m ascii_vision.cli photo.jpg --auto-orient --exif -w 200
```

### Пакетная обработка
```bash
python -m ascii_vision.cli --batch "test_img/*.png" --binary --invert
```

## 3. Флаги

| Флаг | Назначение |
|------|-----------|
| `--binary` | Бинаризация: ч/б без градиентов — **для схем и текста** |
| `--invert` | Светлый фон + тёмные символы (привычно для LLM) |
| `--multi 80 150` | Несколько размеров: мелкий = форма, крупный = детали |
| `--colors` | Цветовая карта: сетка усреднённых цветов + легенда |
| `--annotate` | OCR-подписи справа от строк ASCII-арта |
| `--ocr` | Блок распознанного текста перед ASCII-артом |
| `--ocr-only` | Только OCR, без ASCII |
| `--ocr-lang rus+eng` | Язык распознавания |
| `--color-grid 30` | Разрешение цветовой сетки |
| `--threshold 150` | Порог бинаризации (по умолчанию — авто-порог Otsu) |
| `--levels 16` | Количество уровней градации (по умолчанию 10, макс. 70) |
| `-a 0.5` | Коррекция пропорций (по умолчанию 0.75, для терминала 0.5) |
| `--auto-orient` | Авто-поворот по EXIF-ориентации |
| `--exif` | Извлечь метаданные EXIF в отдельную секцию |
| `--crop X,Y,W,H` | Обрезать область перед обработкой |
| `--batch` | Пакетная обработка по glob-шаблону |
| `--overwrite` | Перезаписать output.md (без timestamp в имени) |
| `--progress` | Показывать прогресс обработки |
| `--no-dir` | Вывод в stdout вместо сохранения в папку |
| `-o путь` | Явный путь для сохранения |

## 4. Структура вывода

Файл `output.md` содержит секции:
- **EXIF Metadata** (если `--exif`) — метаданные изображения
- **OCR Text** — распознанные надписи
- **Color Map** — цветовая сетка и легенда
- **ASCII Art** (одна или несколько ширин) — визуальное представление
- **Аннотации** (если `--annotate`) — подписи у строк

## 5. Как анализировать результат

1. Прочитай OCR-текст — получи список надписей
2. Изучи Color Map — определи цветовые зоны
3. Посмотри ASCII малой ширины — оцени общую композицию
4. Перейди к большей ширине — рассмотри детали блоков и связей
5. Сформулируй: что изображено, какие элементы, как связаны
6. **Если изображение — диаграмма/схема**: сохрани расшифровку в `{папка_изображения}/{имя}_scheme.md`

Подробная инструкция для LLM по интерпретации ASCII-арта: см. `LLM_PROMPT.md`.

## 6. Формат расшифровки схемы (`_scheme.md`)

Если распознана схема, диаграмма, блок-схема или архитектура — сохрани файл `{имя}_scheme.md` в ту же папку, где лежит `output.md`. Формат:

```markdown
# Название схемы — {имя файла}

## Общее описание
[1-2 предложения: что изображено, назначение схемы]

## Mermaid-диаграмма
```mermaid
flowchart TB
    ...
```

## [Послойная структура / Компоненты / Алгоритм]
| Компонент | Назначение |
|-----------|-----------|
| ...       | ...       |

## Потоки данных / Связи
[описание связей между компонентами]

## Ключевые правила (если есть)
[правила из легенды диаграммы]

## Привязка к строкам ASCII
| Строки | Содержание |
|--------|-----------|
| 1-10   | ...       |
```

**Требования к Mermaid-диаграмме:**
- Используй `flowchart TB` (сверху-вниз) или `flowchart LR` (слева-направо) в зависимости от ориентации схемы
- Группируй блоки в `subgraph` по слоям/назначению
- Для ветвлений (ромбы) используй `{текст}` в имени узла
- Неактивные/отключённые элементы помечай ❌ и пунктирной связью `-.->`
- Старайся восстановить реальные названия компонентов, исправляя ошибки OCR

## 8. Ограничения

- Точность OCR зависит от качества и ориентации текста на изображении
- Цветовая сетка усредняет цвета — тонкие цветные линии могут потеряться
- Фотографии людей/объектов: видны силуэт и цветовые зоны, но не мелкие детали
- Для больших изображений (5000+ px) используй `--max-width 500` для ускорения
- PDF требует установки `pdf2image`: `pip install pdf2image`
