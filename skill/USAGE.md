# ASCII_vision — конвертация изображений в ASCII-арт для LLM

```
ascii-vision <изображение> [флаги]
```

## Основные флаги

| Флаг | Описание |
|------|----------|
| `-w N` | Ширина в символах (по умолчанию — исходная ширина изображения) |
| `-o файл` | Явный путь сохранения (по умолчанию — авто-папка рядом с изображением) |
| `--multi 80 150 300` | Несколько размеров в одном выводе |
| `--max-width N` | Ограничить авто-ширину сверху |
| `--batch` | Пакетная обработка: image = glob-шаблон |
| `--overwrite` | Перезаписать output.md (без timestamp) |
| `--progress` | Показывать прогресс обработки |

## Режимы отображения

| Флаг | Эффект |
|------|--------|
| `--binary` | Бинаризация: только ч/б, без градиентов — **идеально для схем/текста** |
| `--threshold N` | Порог бинаризации 0-255 (по умолчанию — Otsu auto) |
| `--invert` | Светлый фон + тёмные символы — привычно для LLM |
| `-b` / `--blocks` | Unicode-символы ` ░▒▓█` вместо ASCII `@%#*+=-:.` |
| `--levels N` | Количество уровней градации (по умолчанию 10, макс. 70) |
| `-a 0.5` | Коррекция пропорций (по умолчанию 0.75, для терминала — 0.5) |

## OCR и аннотации

| Флаг | Эффект |
|------|--------|
| `--annotate` | Разместить OCR-подписи справа от строк ASCII-арта |
| `--ocr` | Добавить блок распознанного текста перед ASCII-артом |
| `--ocr-only` | Только OCR, без ASCII-арта |
| `--ocr-lang rus+eng` | Язык распознавания |

## Цвета

| Флаг | Эффект |
|------|--------|
| `--colors` | Добавить цветовую карту: сетку и легенду усреднённых цветов |
| `--color-grid 30` | Разрешение сетки в столбцах (по умолчанию 20) |

**Цвета:** W=белый K=чёрный R=красный G=зелёный B=синий Y=жёлтый C=голубой M=пурпурный O=оранжевый P=фиолетовый L=салатовый D=тёмно-серый S=серый N=коричневый I=розовый T=бирюзовый

## Изображение

| Флаг | Эффект |
|------|--------|
| `--auto-orient` | Авто-поворот по EXIF-ориентации (для фото с телефона) |
| `--exif` | Извлечь метаданные EXIF в отдельную секцию вывода |
| `--crop X,Y,W,H` | Обрезать область перед обработкой (левый верхний угол, ширина, высота) |

## Ввод

| Способ | Описание |
|--------|----------|
| `ascii-vision image.png` | Обычный файл |
| `ascii-vision - < image.png` | Из stdin |
| `cat image.png \| ascii-vision -` | Через пайп |
| `ascii-vision --batch "*.png"` | Пакетная обработка |
| `ascii-vision document.pdf` | PDF (требуется pdf2image) |

## Формат вывода

| Флаг | Эффект |
|------|--------|
| `--md` | Markdown-блоки (по умолчанию) |
| `--no-md` | Plain text |

## Примеры

```bash
# Схема/диаграмма — лучший режим
ascii-vision diagram.png --binary --invert --multi 80 150 --annotate -o out.md

# Фотография — детальный просмотр
ascii-vision photo.jpg --invert -w 300 -o out.md

# Фотография с авто-ориентацией и EXIF
ascii-vision photo.jpg --auto-orient --exif --invert -w 200

# Только текст со схемы
ascii-vision diagram.png --ocr-only

# Unicode-блоки для плавных градиентов
ascii-vision photo.jpg --invert --blocks -w 200 -o out.md

# Пакетная обработка всех PNG в папке
ascii-vision --batch "test_img/*.png" --binary --invert

# Из stdin
cat image.png | ascii-vision - --binary --invert
```
